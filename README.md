# Computer_games_2025_26



## Evaluation
- Summative
- Empirical evaluation through user participation
    - Observational evaluation
    - Interviews, Questionaires
- Laboratory study
- Choice of Participents:
    - Should have played some computer games, and are familiar with WASD controls
    - Possibly broad choice of participants skills (might not be possible at all)
- Variables:
    - Independant:
        - Tickloop on off
    - Dependant:
        - Deaths
        - Time to completion
        - Questionaire
- Hypothesis:
    0. Users will use the UI Elements to imporove their gameplay experience and skill
    1. Users will not depend on the UI Elements in any way
- Boxplots, and test if data is a normal distribution. If yes, maybe evaluate using statistical significance
- Since we do not have a lot of participants, within-subject will be choosen. Each player will play both versions (UI on, UI off), choosing one randomly as the start condition (50/50). All dialogue times MUST be subtracted, to allow for better comparisions. After first game completion, participate in the first questionaire. 
- Evaluation is logged by one of the evaluators, but think aloud strategy is not enforced
- Protocol:
    - Computer logging
    - Paper and Pencil
    - Swap each session who is taking the protocol (and who supportes the player)
- Users will explore the game freely
- Pilot testing (19.12, or later if we need more content)
- Location:
    - Lab / Online (must have PC with either Windows or Macos)
- Privacy:
    - Each participant receives a random id
    - Consent form 
- When to stop:
    - Time and participant constrained
